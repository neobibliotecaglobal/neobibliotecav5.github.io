Omitir el bloque else

Python no requiere un bloque else al final de una cadena if-elif. A veces, un bloque else es útil. Otras veces, es más claro utilizar una sentencia elif adicional que capture la condición específica de interés:

age = 12

if age < 4:
    price = 0
elif age < 18:
    price = 25
elif age < 65:
    price = 40
elif age >= 65:
    price = 20

print(f"Your admission cost is ${price}.")

El bloque final elif asigna un precio de 20 $ cuando la persona tiene 65 años o más, lo que es un poco más claro que el bloque general else. Con este cambio, cada bloque de código debe superar una prueba específica para poder ejecutarse.

El bloque else es una sentencia catchall. Iguala cualquier condición que no haya sido igualada por una prueba específica de if o elif, y eso a veces puede incluir datos no válidos o incluso maliciosos. Si tienes una condición final específica que estás comprobando, considera la posibilidad de utilizar un bloque elif final y omitir el bloque else. Como resultado, estarás más seguro de que tu código se ejecutará sólo en las condiciones correctas.
