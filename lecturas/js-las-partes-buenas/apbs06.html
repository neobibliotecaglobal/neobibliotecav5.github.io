Declaraciones sin bloque

Una sentencia if o while o do o for puede adoptar una forma de bloque o de sentencia única. La forma de sentencia única es otra atractiva molestia. Ofrece la ventaja de ahorrar dos caracteres, una ventaja dudosa. Oscurece la estructura del programa, de modo que los manipuladores posteriores del código pueden introducir errores fácilmente. Por ejemplo:

if (ok)
    t = true;

puede convertirse en

if (ok)
    t = true;
    advance(  );

que se parece a:

if (ok) {
    t = true;
    advance(  );
}

pero que en realidad significa:

if (ok) {
    t = true;
}
advance(  );

Los programas que parecen hacer una cosa pero en realidad hacen otra son mucho más difíciles de hacer bien. Un uso disciplinado y coherente de los bloques hace que sea más fácil hacerlo bien.
