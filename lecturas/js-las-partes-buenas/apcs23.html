Expresiones regulares

Las expresiones regulares se escriben en una notación escueta y críptica. JSLint busca problemas que puedan causar problemas de portabilidad. También intenta resolver las ambigüedades visuales recomendando el escape explícito.

La sintaxis de JavaScript para los literales de expresiones regulares sobrecarga el carácter /. Para evitar ambigüedades, JSLint espera que el carácter que precede a un literal de expresión regular sea un carácter ( o = o : o ,.
