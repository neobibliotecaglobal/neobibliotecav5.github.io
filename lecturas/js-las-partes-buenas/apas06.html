tipode

El operador typeof devuelve una cadena que identifica el tipo de su operando. Por tanto:

typeof 98.6

produce 'number'. Desgraciadamente:

typeof null

devuelve 'object' en lugar de 'null'. Uy. Una prueba mejor para null es simplemente

my_value === null

typeof no puede distinguir entre null y los objetos, pero tú sí puedes porque null es falso y todos los objetos son verdaderos:

if (my_value && typeof my_value === 'object') {
    // my_value is an object or an array!
}

Consulta también las secciones posteriores"NaN" y"Matrices falsas".

Las implementaciones no se ponen de acuerdo sobre el tipo de objetos de expresión regular. Algunas implementaciones informan de que:

typeof /a/

es 'object', y otras dicen que es 'function'. Podría haber sido más útil informar de 'regexp', pero la norma no lo permite.
